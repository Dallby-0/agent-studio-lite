# 贡献指南

感谢你考虑为本项目做出贡献！以下是一些指导原则。

## 如何贡献

### 报告问题
- 使用 GitHub Issues 报告 bug 或提出功能建议
- 在创建 issue 前，请先搜索是否已有相关 issue
- 清晰描述问题，包括复现步骤和环境信息

### 提交代码(这个是ai生成的，具体之后再说)
1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 创建 Pull Request

### 开发流程
- 遵循现有的代码风格
- 确保所有测试通过
- 更新相关文档
- 提交信息使用英文，清晰描述更改内容

### 代码规范
- 保持代码简洁和可读性
- 添加必要的注释
- 编写单元测试覆盖新功能

## Pull Request 流程
1. 确保分支基于最新的 master 分支
2. 通过所有 CI 检查
3. 需要至少一名维护者审查通过
4. 维护者合并 PR

## 提问和建议
欢迎通过 issue 讨论任何想法或问题！
